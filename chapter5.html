Chapter 5 · The culmination

Can an agent do this for me?

The last scale isn’t a length — it’s the human one. Four chapters of expert hand-work, handed to a team of AI agents that set up, run and connect the simulations themselves.

Idea  multi-agent ICME orchestration Agents  VASP · LAMMPS · DAMASK Status  VASP agent ~60% · others in progress

By the end of Chapter 4 I had built, by hand, a machine for turning electrons into the stress–strain curve of a real metal. It worked. It was also slow, fragile and almost entirely locked inside my own head — which k-point grid, which potential, which DAMASK flag, which queue on which cluster. I kept thinking: if I can write down the rules I follow, why am I the one following them at 2 a.m.?

Chapter 5 is my attempt to put those rules into software that can act. Not a script — scripts break the moment the input changes — but a set of agents that take a request in plain English, decide what calculation it implies, set it up correctly, run it on the HPC cluster, recover from the errors that always happen, and pass the results down to the next scale.

The architecture

A master supervisor reads your request and routes it to one of three specialist agents — one each for VASP, LAMMPS and DAMASK, the three codes from Chapter 4. Between them sit scale-bridge modules that do the unglamorous, error-prone work of handing parameters from one scale to the next with the units converted and the provenance recorded.

What it feels like to use

The whole point is to collapse days of expert setup into a sentence. The honest version of the demo looks like this — one request, and the agents fan out, run on the cluster, and report back:

icme> compute the elastic constants of FCC aluminium and validate
[master] intent = elastic_constants · material = Al(fcc) · route → VASP agent
[vasp] auto-converging ENCUT + k-points … done (520 eV, 15×15×15)
[vasp] self-healing: recovered 1 ZBRENT error automatically
[vasp] submitting to cluster (PBS) … 6 strained cells … collecting
[result] C11 = 107.3 C12 = 60.9 C44 = 28.8 GPa
[check] vs Materials Project reference … < 1% on all constants ✓
icme> _

You spend four years learning to do this by hand. Then it runs in one sentence — at least, for the parts that already work.

What actually works today — and what doesn’t

I want to be precise here, because the temptation to oversell an AI project is enormous and I would rather you trusted the small true claim than doubted a big one. The VASP agent is real and validated. The LAMMPS and DAMASK agents exist as architecture and partial workflows, and the full three-code chain is not yet validated end to end on titanium. This is a proof of concept with momentum, not a finished product.

Electronic structure, elastic constants, GSFE, phonons (VASP) · temperature-dependent elasticity, dislocation dynamics, potential fitting (LAMMPS) · stress–strain, texture, fatigue, forming-limit diagrams (DAMASK).

Underneath, the parts I am proudest of are the unglamorous ones: self-healing error recovery that recognises 13 VASP and 8 LAMMPS failure signatures and fixes them, automatic convergence testing, a structure-memory cache so the system never re-runs a calculation it has already done, and a provenance table that tags every number with where it came from — computed or literature. Those are the things that separate an agent from a fragile script.

The roadmap: finish the LAMMPS dislocation-mobility and potential-fitting workflows; complete DAMASK texture and forming-limit workflows; validate the full VASP→LAMMPS→DAMASK chain on titanium at the cryogenic temperatures of Chapter 4; then extend beyond pure metals to the Ti–Al and Ti–V alloys of Chapter 1. That last step would close the loop — the agent re-deriving, automatically, the very results this thesis was built on.

Looking back along the thread

Five chapters, one idea. I started by asking how strong a single bond is, and built a parameter-free strength predictor. I asked which way a bond points, and predicted slip across the whole HCP family. I asked whether two elements will mix, and turned eighty-year-old intuition into twelve interpretable numbers. I asked whether any of it holds in a real, cold piece of titanium — and built the pipeline, and the experiment, to check. And finally I asked whether a machine could do all of it, and started teaching one to.
